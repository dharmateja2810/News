## **API Endpoints**

### **Authentication**
```
POST   /api/auth/register     Register new user
POST   /api/auth/login        Login user
```

### **Users**
```
GET    /api/users/me          Get current user
PATCH  /api/users/theme       Update theme preference
```

### **Articles**
```
GET    /api/articles                Get all articles (paginated)
GET    /api/articles/:id            Get single article
GET    /api/articles/categories     Get all categories
POST   /api/articles                Create article (n8n webhook)
```

### **Bookmarks**
```
GET    /api/bookmarks               Get user's bookmarks
POST   /api/bookmarks/:articleId    Toggle bookmark
POST   /api/bookmarks/like/:articleId   Toggle like
```

## **Security**

### **Authentication**
- JWT tokens with Bearer scheme
- Password hashing with bcrypt (10 rounds)
- Token expiration: 7 days (configurable)

### **Authorization**
- Protected routes use `@UseGuards(JwtAuthGuard)`
- User context available in `@Request() req`
- n8n webhook protected by secret header

### **Validation**
- DTO validation with `class-validator`
- Request body validation
- Query parameter validation

### **CORS**
- Configurable allowed origins
- Credentials support enabled

## **Scalability Considerations**

### **Current Setup (MVP)**
- Single PostgreSQL instance
- No caching layer
- Direct database queries

### **Future Improvements**
1. **Redis Caching**
   - Cache frequently accessed articles
   - Cache user sessions
   - Reduce database load

2. **CDN for Images**
   - CloudFront/Cloudflare
   - Reduce bandwidth costs
   - Faster image loading

3. **Rate Limiting**
   - Protect against abuse
   - Per-user limits
   - Per-endpoint limits

4. **Database Optimization**
   - Read replicas for queries
   - Connection pooling
   - Query optimization

5. **Horizontal Scaling**
   - Multiple API instances
   - Load balancer
   - Stateless design (JWT)
